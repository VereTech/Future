Service Description:

Monitors the status of the hardware sensor, as well as how many times the sensor has reported a Warning or Failed status.

Notes

Configuration

To configure this service, you will have to specify the name of the sensor to be monitored. For a list of sensors, query the ProCurve with the 1.3.6.1.4.1.11.2.14.11.1.2.6.1.7 SNMP OID.

Errors

Misconfigured on the dashboard probably means that either the wrong SNMP community string has been specified for the device, or that the service has been configured with a non-existent sensor name.

Comments

The values for Status are mapped as follows:

 

A value of 0 (Normal) in N-central means the drive is reporting a value of 4

A value of 1 (Warning) in N-central means the drive is reporting a value of 1 or 3

A value of 2 (Failed) in N-central means the drive is reporting a value of 2 or 5

 

Here are the text descriptions for each value:

 

1: Unknown

2: Bad

3: Warning

4: Good

5: Not Present
