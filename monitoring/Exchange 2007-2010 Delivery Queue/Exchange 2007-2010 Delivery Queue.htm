Overview

Service Name:

Exchange 2007/2010 Delivery Queue

Service ID:

24189

Service Type:

Windows Servers

Supported Devices / OS:

Any WMI-enabled Windows Server running either Exchange 2007 or Exchange 2010.

Maximum Service Instances / Device:

100

Time to Stale:

30

Service Description:

Monitors delivery queues on an Exchange 2007/2010 server. This service looks in the root\cimv2 WMI namespace for the Win32_PerfRawData_MSExchangeTransportQueues_MSExchangeTransportQueues WMI class. It pulls the following properties:

 

ActiveMailboxDeliveryQueueLength

ActiveNonSmtpDeliveryQueueLength

ActiveRemoteDeliveryQueueLength

AggregateDeliveryQueueLengthAllQueues

LargestDeliveryQueueLength

PoisonQueueLength

RetryMailboxDeliveryQueueLength

RetryNonSmtpDeliveryQueueLength

RetryRemoteDeliveryQueueLength

SubmissionQueueLength

UnreachableQueueLength

Notes

Configuration 

You must specify the name of the Queue to be monitored. For a list of the queues on a specific device, look at Name property of each instance in the Win32_PerfRawData_MSExchangeTransportQueues_MSExchangeTransportQueues WMI class, under the root\cimv2 WMI namespace.

Errors

Misconfigured on the dashboard probably means that an invalid Queue has been specified, or that a WMI communication issue is occurring.
